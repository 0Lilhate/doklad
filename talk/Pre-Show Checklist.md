# Pre-Show Checklist — за 30 минут до выступления

Жёсткий механический чек-лист. Не пропускать пункты, не делать "по памяти", не делать "пробежимся быстро". Если на любом пункте что-то не так — стоп, чинить, не идти на сцену с открытой проблемой.

## T-30 — техника

### Hardware
- [ ] Ноутбук на полной зарядке + кабель в розетке
- [ ] HDMI/Thunderbolt-адаптер для проектора подключён и работает
- [ ] Внешний монитор/проектор показывает то же, что и экран ноутбука (mirror, не extend — иначе demo на не том экране)
- [ ] Звук: проверен микрофон, проверен output (если что-то воспроизводится)
- [ ] Кликер для слайдов проверен (если используется)
- [ ] Запасной USB-стик с offline-копией слайдов и fallback-артефактов

### Software
- [ ] opencode запущен, последняя версия
- [ ] opencode авторизован, не просит пароль/токен
- [ ] MCP-сервер для OpenAPI поднят, opencode видит его (тест-промпт прошёл)
- [ ] **Реальный demo-сервис** открыт в IDE на чистой ветке (например, `talk-demo`)
- [ ] Сборка зелёная (последний прогон < 1 часа назад)
- [ ] git status чистый (никаких `M` файлов от прошлой репетиции; все эксперименты сброшены)
- [ ] Папки `~/.aws`, `~/.ssh`, секретные `.env` НЕ открыты ни в одном окне IDE / терминала
- [ ] AGENTS.md в demo-репозитории актуален (если используется в demo)
- [ ] Slides открыты, режим презентации проверен
- [ ] Браузер: incognito или новый профиль, нет залогиненных сессий с production-системами

### Network
- [ ] Wi-Fi работает, скорость ≥ 5 Mbps (тест: speed test или загрузка опера)
- [ ] Mobile hotspot готов как backup (если Wi-Fi упадёт)
- [ ] VPN ВЫКЛЮЧЕН (если используется корпоративный — он может блокировать opencode endpoints)
- [ ] DNS resolves: пробный запрос к opencode-эндпоинту прошёл

## T-25 — Demo dry-run на реальном сервисе

Полный прогон 12 шагов на **реальном сервисе** (не на synthetic rig — теперь у нас realname-режим) **без зала**. Если хоть один шаг работает не так, как должен — фиксим или переключаемся на fallback для этого конкретного шага.

**Внимание:** все эксперименты этого dry-run после прогона должны быть откачены через `git stash` или `git restore .` — иначе на сцене будешь работать на грязном дереве.

- [ ] Шаг 1: `demo/ticket.md` (бизнес-формулировка PKG-2026-104, без code refs) открывается
- [ ] Шаг 2: первый промпт аналитика получает релевантный ответ за < 30 сек, **без упоминания имён классов / endpoint'ов**
- [ ] Шаг 3: вопросы к бизнесу включают тот, который мы заранее заложили как "missed question" — что показывать при истёкшем grace (см. Live Demo Script)
- [ ] Шаг 4: ФС обновляется после ответов бизнеса, edge cases явные в бизнес-терминах
- [ ] Шаг 5: Business TO-BE Mermaid рендерится в preview, участники только «Пользователь / Клиент / API» (нет `Ctrl`, `Svc`, `QB`)
- [ ] Шаг 6: переключение на dev-сессию opencode (MCP к коду доступен)
- [ ] Шаг 6: impact area показывает 3+ файла с реальными file:line, найденных через MCP
- [ ] Шаг 7: технический AS-IS Mermaid sequence diagram из кода — рендерится в preview
- [ ] Шаг 8: план реализации возвращается **без кода** (если агент сразу пишет код — нужна корректировка промпта)
- [ ] Шаг 9: реализация — diff < 50 строк в 2 production-файлах
- [ ] Шаг 10: 4 теста — happy path + expired (null) + null end + same-day (0), mvn зелёный
- [ ] Шаг 11: self-review находит **truncation-bug** (ChronoUnit.DAYS на ZonedDateTime) или подсаженный
- [ ] Шаг 11b: после fix второй mvn-прогон зелёный
- [ ] Шаг 12: MR description содержит все секции из шаблона (включая раздел Риски с truncation-историей)

> Если хоть один пункт failed — отметить и подготовить fallback-артефакт для этого шага. Не пытаться "по горячему чинить" во время доклада.

- [ ] Уведомления выключены: Do Not Disturb на macOS / Focus Mode на Windows
- [ ] Окна с реальным кодом закрыты в IDE (recent files почищены, см. ниже)
- [ ] Терминал: `history -c` или новая сессия. `clear` всех скроллбэков

- [ ] Wallpaper рабочего стола нейтральный (не корпоративный, без личных фото с распознаваемыми элементами)
- [ ] Ярлыки на рабочем столе — без названий реальных проектов
- [ ] `~/Documents`, `~/Downloads` папки — на всякий случай не открывать
- [ ] Файловый менеджер: navigation history очищен или browser-style новый
- [ ] `.env` / `application.yml` в demo-rig — synthetic values only
