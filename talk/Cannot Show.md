## КАТЕГОРИЧЕСКИ НЕЛЬЗЯ показывать

### 1. Секреты и учётные данные
- Токены (JWT, OAuth, API keys), даже expired, даже «тестовые»
- Пароли, ключи (private keys, certificates)
- Строки подключения к БД с credentials
- Содержимое `.env`, `application-prod.yml`, `secrets.yaml`
- IP-адреса production-инфраструктуры с открытыми портами

### 2. Персональные данные клиентов
- Реальные ФИО, паспортные данные, ИНН, СНИЛС, телефон, email
- Любые финансовые данные с реальными суммами и счетами
- Реальные ID клиентов, даже UUID
- Сэмпл-данные из production-БД (могут содержать чувствительное)

### 3. NDA-документы и IP
- Полные тексты внутренних архитектурных документов / регламентов / DPA
- Алгоритмы скоринга, ценообразования, специфические бизнес-правила, составляющие конкурентное преимущество
- Документы партнёров / контрагентов

### 4. Чужой код
- Код других команд / partner integrations, к которому нет твоего operational ownership'а
- Код, на показ которого не получено явного разрешения от владельца сервиса

### 5. Production-логи с чувствительным
- Логи с реальными ID операций, клиентскими данными, security events
- Полные stack-trace'ы с прод (могут содержать env vars, paths)
